Вычислить общий недублированный охват и частоту (TURF)

Эта процедура вычисляет значения TURF для набора переменных ответа. TURF отличается от использования простых частот учетом перекрывающихся ответов.

Охват для сочетания переменных - это число наблюдений, в которых по крайней мере одна переменная в сочетании имеет положительный отклик. Частота - это общее число положительных откликов для сочетания. При вычислении процента откликов используются все положительные отклики для любой переменной в таблице в качестве знаменателя.

Переменные для анализа Выберите переменные для анализа. Все они должны быть кодироваться одним способом, по крайней мере, для положительных ответов.

Максимальное число сочетаний переменных Задайте максимальное число сочетаний, для которых нужно вычислить TURF. Если задать, например, 3, будут показаны наилучшие значения для всех индивидуальных переменных, всех сочетаний двух переменных и всех сочетаний трех переменных.

Число сочетаний для вывода Задает максимальное число строк в каждой таблице. Строки сортируются в порядке убывания уровней отклика с нарушением привязок по частоте. Однако возможны дополнительные привязки к последнему элементу в таблице.

Минимальный процент положительных откликов Если процент положительных откликов для переменной меньше этого значения, он будет исключен из анализа. Переменные с нулевым количеством положительных откликов никогда не используются в анализе. Этот фильтр не применяется к принудительно включаемым переменным: такие переменные включаются всегда.

Положительные значения отклика По умолчанию положительный отклик задается значением 1. Введите одно или несколько значений, чтобы задать положительные отклики иначе. Например, если значения откликов варьируют от 1 до 5, можно задать максимальный уровень, введя 5 или 4 5. Между значениями нужно ввести пробел.

Число обязательных переменных анализа Это значение, n, указывает, что первые n переменных, перечисляемые в окне Переменные для анализа, обязательны, и таким образом, будут появляться в любом сочетании переменных, независимо от их эффекта для охвата. Если это значение - 0, никакие переменные не включаются принудительно. Если это значение положительно, циклы исключения невозможны.

Число удалений наилучшей переменной и повторений анализа Укажите, сколько раз должна удаляться наилучшая отдельная переменная с повтором анализа для оставшихся переменных. Используются только случаи, когда наилучшая переменная не дала положительного отклика. Эффект является кумулятивным, поэтому в ходе двух циклов анализ будет повторен без лучшей переменной в первом цикле и лучшей переменной во втором цикле. Будут использоваться только наблюдения с отрицательными значениями обеих переменных во втором цикле.

В случае использования расширенного отбора цикл удаления применяется только к финальному раунду

График Охват и частота Включите этот переключатель, чтобы создать график, содержащий наилучшие охват и частоту для каждого размера группы.

Показать только результаты для максимального сочетания переменных Включите этот переключатель, чтобы показать только таблицу для максимального сочетания переменных. Если этот переключатель не включен, таблицы выводятся для наилучших сочетаний для каждого размера до максимального числа переменных включительно. Если используется расширенный отбор, выводится по одной таблице для каждого поднабора расширенного отбора, независимо от этого параметра.

Предельный размер В анализе TURF может потребоваться вычислить неожиданно большое количество объединений наборов, что требует значительного времени. Если задать предельное значение, анализ не будет выполняться, если это значение превышено. Вывод TURF включает объявление числа объединений наборов, которые являются обязательными. Можно задать этот предел равным 0, чтобы оценить размер проблемы, не выполняя анализ. Информация о размере содержится в объекте журнала средства просмотра.

Если используется расширенный отбор, размер применяются отдельно к каждому поднабору расширенного отбора и к финальному раунду.

Расширенный отбор

По мере роста числа переменных и числа сочетаний резко возрастает время вычислений. При помощи расширенного отбора удается найти приближенное, но обычно достаточно точное решение за гораздо меньшее время.

Использовать расширенный отбор для больших задач Если этот переключатель включен, процедура будет выполнять расширенный отбор для случайно выбранных поднаборов переменных, причем каждая переменная будет включена в тот или иной отбор, а победители будут включены в финальный раунд. Размер поднабора расширенного отбора и число победителей, включаемых в финалы, регулируются параметрами Размер поднабора расширенного отбора и Число победителей. Включите переключатель Автоматически, что эти параметры были назначены процедурой, или включите переключатель Задать, чтобы задать одно или оба из этих значений.

Значения размера расширенного отбора и числа победителей не учитывают принудительно включаемые переменные, так как они автоматически включаются в каждый расширенный отбор.

Для повышения производительности число победителей должно быть намного меньше размера поднабора расширенного отбора. Лучше всего так сбалансировать параметры расширенного отбора и победителей, чтобы число участников финального раунда не слишком превышало численность предварительных раундов.

Если размер поднабора расширенного отбора достаточно велик, чтобы включить все переменные в один расширенный отбор, предварительный раунд пропускается.

Поскольку те или иные переменные выбираются для расширенного отбора случайным образом, для лучшего понимания того, насколько хорошо решение на основе расширенного отбора по сравнению с точным решением, можно выполнить процедуру несколько раз и изучить наилучший охват, полученный каждый раз, или использовать небольшое увеличение параметра число победителей.

Веса важности

Переменным в анализе можно назначить веса важности, которые не следует путать с весами наблюдений. Они не используются при поиске наилучшего охвата, но вывод содержит некоторые статистики, основанные на них.

Если используются веса, вывод включает среднее значение и общую важность положительных откликов. Вначале для каждого наблюдения, у которого есть хотя бы один положительный отклик, вычисляется среднее значение весов важности для положительных откликов для конкретной комбинации переменных. Вывод содержит среднее из этих средних. Выводится также общая важность.

Проверка весов Эти радиокнопки определяют, как задаются веса и как они проверяются. Если выбрано Все переменные... и используются веса, то спецификация должна включать все переменные в анализе. Если выбрано Предполагается вес..., то вес переменных, не показанных в списке, полагается равным 1.

Спецификации весов Введите имена и веса переменных. После имени каждой переменной следует вес важности для этой переменной. Например,
V1 2 V2 1.5
назначает вес 2 переменной V1 и вес 1.5 переменной V2. В качестве десятичного разделителя всегда используется точка.

Примечания к вычислениям

При увеличении максимального размера группы результаты повторного анализа связываются с результатами, показанными в таблицах без повторения. Однако в некоторых ситуациях возможны различия. Например, из лучшей группы из двух переменных может быть исключена лучшая одиночная переменная. В повторном анализе эти различия неощутимы, но они выявляются методом, использующим несколько групп. Однако использование циклов удаления может сделать слабый эффект от добавления переменных более четким. При использовании циклов удаления может быть полезно задать максимальный размер группы и число показываемых комбинаций равным 1.

Результаты, выраженные в процентах, зависят от использованных наблюдений, поэтому база будет разной при использовании двух подходов. Работа с пороговыми значениями может также привнести различия, поскольку зависит от базы.

База процентов охвата включает все отклики (после выполнения фильтрации), поэтому пропущенные значения могут вносить вклад в базу. Если группа содержит несколько переменных, для ее расчета нет другого очевидного способа.

Вычислительные затраты на эти расчеты очень быстро увеличиваются с ростом числа переменных и значения Максимальное число комбинаций переменных. Вывод содержит число необходимых расчетов объединений наборов. Для больших задач рекомендуется использовать поднаборы расширенного отбора.

Все данные о положительных откликах хранятся в памяти. Требуется только один проход по данным.
